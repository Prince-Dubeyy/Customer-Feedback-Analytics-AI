# Root Cause Analysis: Sentiment Classification Failures

## Overview
An audit of the current sentiment classification pipeline (`01_data_cleaning.py`, `04_model_training.py`, and `ml_service.py`) reveals several fundamental flaws in data quality, preprocessing, and model configuration that directly contribute to poor generalization and Negative prediction bias.

## 1. Negative Prediction Bias & Neutral Class Weakness
- **Issue**: The model defaults to Negative on unseen reviews, and rarely predicts Neutral correctly unless the review exactly matches a few specific phrases.
- **Root Cause**: The training dataset only contains ~1,500 records. Crucially, the 500 "Neutral" records are completely synthetic—generated by copying exactly 10 short sentences 50 times over. This means the model's entire understanding of "Neutral" is limited to a vocabulary of less than 40 words. When a real, unseen review is evaluated, it has zero TF-IDF overlap with the Neutral class, forcing the model to arbitrarily pick between Positive and Negative. Negative language in the dataset is highly polarized and distinct, making it an easy "catch-all" for unrecognized text.

## 2. Poor Generalization
- **Issue**: Misclassification of obvious real-world reviews.
- **Root Cause**: The dataset size (1,500 records) is vastly insufficient for real-world NLP. The vocabulary diversity is too low. A production model requires tens of thousands of varied examples to learn nuances like sarcasm, mixed sentiments, or domain-specific language.

## 3. Preprocessing Flaws
- **Issue**: Preprocessing does not strip URLs, mentions (@), or hashtags (#), which inject noise into the TF-IDF vectorizer.
- **Root Cause**: `app.core.text_preprocessing` only removes HTML and non-alphabetic characters. It leaves noise that can skew feature weights.

## 4. Suboptimal Model Architecture
- **Issue**: The TF-IDF Vectorizer is configured with `ngram_range=(1,1)`.
- **Root Cause**: Using only unigrams completely destroys local context. For example, the phrase "not good" is split into "not" and "good". "Good" heavily weights toward Positive, while "not" is neutral/negative. The model lacks bigram/trigram understanding, which is essential for capturing negations.

## Conclusion
To achieve production quality, we must immediately replace the 1,500-row synthetic dataset with a massive (15,000+ row) open-source dataset, introduce bigram vectorization, and rigorously benchmark Logistic Regression against SVM and Naive Bayes.
